# Vibepollo 1.18.1-beta.1 - 2026-07-15

Notice: Vibepollo may trigger false-positive antivirus alerts.

## Changes
- Added adaptive smoother capture for virtual displays: the recommended mode now uses normal refresh on the desktop and 4x refresh while a game is active, with disabled automation and legacy fixed-2x modes also available.
- Added per-client virtual-display HDR peak calibration from 400 to 2000 nits and per-client Windows scaling, keeping display identity, HDR profiles, and encoder metadata aligned with each client.
- Reduced virtual-display stream startup latency by skipping redundant HDR transitions and settle waits on verified displays while retaining the full recovery path when needed.

## Fixes
- Prevented Playnite lifecycle timing and launcher process detection from disconnecting healthy game streams.
- Respected disabled HDR display handling when creating or recovering virtual displays instead of enabling Windows HDR from raw client intent.
- Restaged changed virtual-display driver payloads during upgrades, removing mixed stale packages and reviving the device without requiring a reboot.
- Recovered stream startup when RTSS hooks stopped responding by timing out stalled calls and falling back to the NVIDIA frame limiter.
- Preserved temporary per-client displays through system sleep and recreated missing leased displays after a headless resume.
- Restored RTX HDR brightness above 1000 nits and kept initial and live HDR peak metadata synchronized for native NVENC HEVC and AV1 streams.
